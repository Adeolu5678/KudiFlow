# Acceptance Criteria Checklist

## App

- [ ] App builds in Xcode.
- [ ] App runs on simulator/device.
- [ ] Firebase initializes.
- [ ] No crash on launch.

## Navigation

- [ ] Home tab works.
- [ ] Log tab works.
- [ ] Assistant tab works.
- [ ] Ledger tab works.
- [ ] Credit tab works.

## Ledger

- [ ] Manual entry can be saved.
- [ ] Firestore receives entry.
- [ ] Ledger list displays saved entry.
- [ ] Dashboard updates after save.

## AI extraction

- [ ] Text extraction works.
- [ ] Image extraction works or mock fallback works.
- [ ] JSON parse failure handled.
- [ ] Low confidence warning shown.
- [ ] Review screen appears before save.

## Assistant

- [ ] "Who owes me?" works.
- [ ] "How is my business today?" works.
- [ ] "Write reminder" works.
- [ ] Assistant does not answer unrelated questions.
- [ ] Assistant uses ledger numbers.

## Credit summary

- [ ] Shows sales.
- [ ] Shows expenses.
- [ ] Shows receivables.
- [ ] Shows estimated profit.
- [ ] Does not promise loan approval.

## Demo

- [ ] Full demo under 3 minutes.
- [ ] Mock fallback available.
- [ ] Seeded data reset available.
- [ ] Team has rehearsed speaking roles.
